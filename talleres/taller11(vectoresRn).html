Taller computacional Vectores en \(\mathbb{R}^n\)#

Objetivo: Utilizar un ejemplo pr√°ctico de procesamiento de texto para visualizar y comprender conceptos fundamentales de √°lgebra lineal como vectores en \(\mathbb{R}^n\), bases, dimensi√≥n e independencia/dependencia lineal.

Ene l procesamiento de texto, en sus inicios se uso la representaci√≥n de documentos como una bolsa de palabras, donde cada documento se representaba como un vector en \(\mathbb{R}^n\) y cada palabra como una dimensi√≥n. En este taller, utilizaremos un conjunto de documentos para ilustrar estos conceptos.

Para realizar esto a cada documento se le hace un preprocesamiento, que consiste en:

  1. Tokenizaci√≥n: Separar el texto en palabras o tokens.

  2. Lematizaci√≥n: Reducir las palabras a su forma base o lema.

  3. Eliminaci√≥n de stopwords: Quitar palabras comunes que no aportan significado (como ‚Äúy‚Äù, ‚Äúel‚Äù, ‚Äúde‚Äù).

  4. Vectorizaci√≥n: Representar cada documento como un vector en \(\mathbb{R}^n\) donde \(n\) es el n√∫mero de palabras √∫nicas en el conjunto de documentos.

  5. Normalizaci√≥n: Escalar los vectores para que tengan una longitud de 1 (opcional).

Ejemplo pr√°ctico#

Supongamos que nuestro documento es el siguiente:

‚Äúla casa es grande y la casa es azul‚Äù

Paso 1: Tokenizaci√≥n#

En la tokenizaci√≥n, separamos el texto en palabras. En este caso, obtenemos:

["la", "casa", "es", "grande", "y", "la", "casa", "es", "azul"]

Paso 2: Lematizaci√≥n#

La lematizaci√≥n consiste en reducir las palabras a su forma base. En este caso, podemos obtener:

["la", "casa", "ser", "grande", "y", "la", "casa", "ser", "azul"]

Paso 3: Eliminaci√≥n de stopwords#

En este paso, eliminamos las palabras que no aportan significado. En este caso, eliminamos ‚Äúla‚Äù, ‚Äúy‚Äù y ‚Äúes‚Äù. Obtenemos:

["casa", "grande", "casa", "azul"]

Paso 4: Vectorizaci√≥n#

Ahora, representamos cada documento como un vector en \(\mathbb{R}^n\). Para esto, creamos un vocabulario con las palabras √∫nicas del documento. En este caso, el vocabulario es:

["casa", "grande", "azul"]

Ahora, representamos el documento como un vector en \(\mathbb{R}^3\) (ya que tenemos 3 palabras √∫nicas):

[2, 1, 1]

Paso 5: Normalizaci√≥n#

Finalmente, normalizamos el vector para que tenga una longitud de 1. La longitud del vector es: $\( \sqrt{2^2 + 1^2 + 1^2} = \sqrt{4 + 1 + 1} = \sqrt{6} \)\( Por lo tanto, el vector normalizado es: \)\( \left[\frac{2}{\sqrt{6}}, \frac{1}{\sqrt{6}}, \frac{1}{\sqrt{6}}\right] \)$

Coseno entre vectores#

El coseno entre dos vectores \(u\) y \(v\) en \(\mathbb{R}^n\) se puede calcular utilizando el producto punto y la longitud de los vectores. La f√≥rmula es:

\[ \cos(\theta) = \frac{u \cdot v}{||u|| \cdot ||v||} \]

Donde \(u \cdot v\) es el producto punto de los vectores \(u\) y \(v\), y \(||u||\) y \(||v||\) son las longitudes de los vectores \(u\) y \(v\), respectivamente. El √°ngulo \(\theta\) se puede obtener utilizando la funci√≥n np.arccos de NumPy, que devuelve el √°ngulo en radianes. Para convertirlo a grados, se puede usar la funci√≥n np.degrees.
